# خريطة الطريق المعمارية (Engineering Action Plan) — مشروع أفان تيتر

> **توطئة هندسية:**
> بناءً على التكوين الحالي لمستودع `src/` (الذي يوظّف React كغلاف للواجهة، و Tiptap كمحرك، و State Machines للتصنيف)، تم تفكيك متطلبات السوق إلى حزم عمل هندسية (Engineering Epics). هذه الخطة تُترجم الأهداف التنافسية إلى مهام برمجية قابلة للجدولة في (Jira/GitHub Projects).

---

## المرحلة 1: الإطلاق التقني (Tech Debt & Missing Core)
تعالج هذه المرحلة الثغرات المعمارية التي تمنع المشروع من التنافس الأساسي (الميزات المفقودة 🔴).

### Epic 1.1: محرك التصدير المتعدد (Multi-format Export Engine)
*   **الوصف:** بناء واجهة خلفية (أو مكتبة متصفح) قادرة على تحويل `ScreenplayBlock` إلى PDF و FDX.
*   **المهمة 1 (PDF):** 
    *   **النهج:** استخدام `puppeteer` المثبت حالياً على السيرفر (أو `pdfmake` للعمل في Client-side) لبناء كلاس `PDFExporter`.
    *   **المواصفات:** يجب حساب ترويسات الصفحات وتذييلاتها (A4 96PPI)، وتمرير بيانات `DocumentStats` للترقيم، والتأكد من قصّ كتلة الـ `Payload` وعدم طباعتها.
*   **المهمة 2 (FDX Export):**
    *   **النهج:** بناء `xmlbuilder` يحول مصفوفة `ScreenplayBlock` إلى عقد `<Paragraph>` الخاصة بتنسيق FDX 11+.
    *   **المواصفات:** تعيين `Type` لكل نوع عنصر لدينا (رأس مشهد = Scene Heading).

### Epic 1.2: منظومة الغلاف (Title Page Generator)
*   **الوصف:** بناء معمارية بيانات للغلاف منفصلة عن نص السيناريو.
*   **التنفيذ المعماري:**
    1.  تحديث `document-model.ts` لإضافة كائن جديد للبيانات الوصفية `DocumentMetadata` (اسم الفيلم، الكاتب، التاريخ).
    2.  تعديل المُشفر `encodeScreenplayPayload` ليتسع ويحفظ هذا الكائن داخل الـ Base64 ليتنقل مع النص.
    3.  بناء مكون React كُـ Modal لحقن هذه البيانات قبل تمريرها لمحرك الطباعة في Epic 1.1.

### Epic 1.3: البحث النمطي المتقدم (Type-Aware Search & Replace)
*   **الوصف:** محرك بحث يفهم سياق الـ Tiptap Nodes.
*   **التنفيذ المعماري:**
    *   بناء إضافة (Tiptap Extension) جديدة باسم `AdvancedSearchReplace`.
    *   توسيع خوارزمية البحث للفلترة عبر حقل `data-type`. 
    *   *مثال بالكود:* دالة تقبل `(query: string, elementFilter: LineType[])` لتبحث عن الكلمة فقط إذا كانت ضمن عقدة من نوع `dialogue`.

## المرحلة 2: تعظيم التفوق الهندسي (Over-Engineering the Core)
تركز هذه المرحلة على تعزيز نقاط قوتنا (الذكاء، التصنيف، دعم المحلية 🟢) وجعلها شبه مستحيلة الاستنساخ للمنافسين.

### Epic 2.1: وكيل الذكاء الاصطناعي التفاعلي (Interactive Agent Review)
*   **الوصف:** تحويل الكود النائم `agent-review.ts` من موجه صامت إلى مساعد تفاعلي.
*   **التنفيذ المعماري:**
    1.  تغيير استجابة السيرفر في الواجهة من "تعديل صامت" (Silent Patch) إلى تسجيل مخرجات Claude داخل واجهة `diff`.
    2.  بناء مكون React (`AgentSuggestionBubble`) يلتصق بالسطر "المشبوه" في Tiptap عبر `tippy.js`، يعرض للمستخدم: *الذكاء الاصطناعي يقترح تحويل هذا السطر إلى حوار، هل توافق؟*.
    3.  **تجريد المزوّد (Provider Abstraction):** تعديل `AgentReview` ليكون واجهة تقبل Claude أو ChatGPT لتفادي الاعتماد الكلي على جهة واحدة (Vendor Lock-in).

### Epic 2.2: الإكمال التلقائي الذكي للعناصر (Semantic Auto-Complete)
*   **الوصف:** تحسين آلة الحالة (State Machine) للتنبؤ المسبق بالمقاصد.
*   **التنفيذ المعماري:**
    *   إنشاء Tiptap Plugin (مثل `SuggestionsPlugin`) يراقب الـ `transaction` الحي.
    *   عند اكتشاف كلمات دلالية (Regex: `^(INT|EXT|داخلي|خارجي)`), يتم فوراً عرض نافذة منسدلة (Popover) بأسماء الأماكن المستخرجة سابقاً من `DocumentStats`، وبمجرد الضغط على Enter يتحول السطر برمجياً `editor.commands.setSceneHeader()`.

### Epic 2.3: واجهة المعايرة (Typing System Settings UI)
*   **الوصف:** ربط خوارزمية الـ 3 أوضاع كتابة بواجهة المُستخدم الميتة.
*   **التنفيذ المعماري:**
    *   بناء نموذج (Form) في الشريط الجانبي يُغذي حالة التطبيق في `App.tsx` المُسماة `typingSystemSettings`.
    *   إنشاء Input Range (من 500ms إلى 3000ms) يُعدّل قيمة الخمول `idleTime` لخاصية الـ `auto-live` ليتيح للمستخدم التسريع أو الإبطاء من وتيرة التصنيف.

## المرحلة 3: التوسعات الإنتاجية (Production Utilities)
إدخال خريطة الميزات المتطلبة للمقالب الأكبر 🟡 (نطاقات المخرجين والإنتاج).

### Epic 3.1: مُستكشف المشاهد (Scene Navigator API)
*   **الوصف:** استخراج شجرة الـ DOM الديناميكية.
*   **التنفيذ المعماري:**
    *   كتابة دالة `useSceneExtractor` كـ Hook في React يتصل بـ `editor.on('transaction')`.
    *   استخراج جميع الـ Nodes المعرّفة كـ `scene-header-1` و `-2` و `-3` وحفظ الفهرس الخاص بها (Index & Offset).
    *   بناء قائمة جانبية (Sidebar Tree)؛ عند النقر على عقدة، يتم استدعاء أوامر التحرير `editor.commands.focus(position)` لإجراء حركات التصفح السريع.

### Epic 3.2: تتبع النسخ المعرّف لونياً (Color-coded Revisions)
*   **الوصف:** إدارة الـ Versions والتعديلات.
*   **التنفيذ المعماري:**
    *   بناء نظام (Mark) جديد في Tiptap تحت اسم `RevisionMark` يحمل Property خاص برقم أو معرف النسخة.
    *   عند تفعيل الكاتب لوضع "المسودة الحمراء"، أي نص جديد يتم إحاطته بـ `<span class="rev-red">`، وأي نص يُحذف، لا يُزال بل يُعطى class `rev-strike`.
    *   يتطلب هذا تعديلاً مهماً وتوسعة في أداة `document-model.ts` والـ `Payload` لضمان تصدير هذه الملاحظات اللونية وعدم تدميرها.

> **خلاصة الجدول الزمني (Sprint Proposal):**
> 1. السبرنت القادم (Sprint 1): إنجاز Epic 1.1 + Epic 1.2 (فك خانقة التصدير لجعله قابلاً للإنتاج).
> 2. السبرنت (Sprint 2): العمل على Epic 1.3 + Epic 3.1 (تحسين أدوات البحث والتصفح).
> 3. السبرنت (Sprint 3): رفاهية الذكاء الاصطناعي وإغراق السوق بالميزات عبر Epic 2.1 و Epic 2.2.
